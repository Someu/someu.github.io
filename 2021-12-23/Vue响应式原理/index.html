Vue响应式原理

发表于 2021-12-23 更新于 2021-12-26

Vue 的响应式，让我们在修改数据的时候，视图也会进行更新。

——自动档，多么美好的世界。

get 和 set 方法

Vue2 通过 Object.defineProperty 方法来代理对象上属性的 get 和 set 操作，而Vue3 中使用 Proxy 的方式处理。两个方式的目的都是一样的，通过代理对象上属性的 get 和 set 方法，实现在获取属性的时候收集依赖，属性发生变更的时候触发监听的更新。下面是Object.defineProperty 和 Proxy 代理对象上属性 get 和 set 方法的案例。

观察者模式

观察者模式又称发布订阅模式，当一个对象（依赖）和多个对象（观察者）之间存在依赖关系时，依赖对象的状态发生改变，观察者对象都会得到通知并且自动更新。观察者模式分为订阅和发布两个环节，订阅环节是观察者在依赖上注册监听，发布环节是依赖状态有变化时通知观察者。下面是一个观察者模式的案例。

Vue 中的响应式

Vue2 中响应式实现可以分为三个环节：

  1. 初始化组件时，将组件的 data 和 props 设置为响应式对象。即为每个属性添加 setter 和 getter 方法，会为每个属性生成一个 dep 实例，该 dep 用于依赖收集和属性值变化时通知相应的 watcher 触发更新。下面是一个简化后的设置对象属性响应式的方法。

数组的处理

Object.defineProperty 可以对对象属性的 get 和 set 操作进行代理，而数组本身存在一些操作方法。Vue 针对这些方法进行了代理，以便在方法调用时响应式更新。

异步更新

异步模式下 watcher 进行更新时，会使用 queueWatcher 将 watcher 推入到一个队列中，该队列的所有任务会在下一个微任务中执行。
